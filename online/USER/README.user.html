Files under USER/ expose our deployment under the final user point of view, the person who wants to run some programs in order to enjoy the services. To comply with that we enumerate the features present in each service as well as we offer a text video, ttyrec recorded, demonstration.

Being all that true, it is also a fact that we take the user point of view as an excuse to expose some details that serve as an introduction to administration technicalities (explained under ADMIN/ directory). This happens specially during ttyrec videos, in fact and, as you already know, we encourage you to imitate them to interact with interesting points of administration work.

1 Important comments

Two aspects you should be familiarized with before continuing are user accounts and ttyrec videos. We offer you some bits about those now:

1.1 Account format

Centralized user accounts that let us use the services: we have created1 two accounts. In memory of the year we studied at a northern Sweden city called Umeå, the names are:

  • umea@casafx.dyndns.org
  • aemu@casafx.dyndns.org (aemu is just the reversed of umea)

Seeing that, a centralized account has the general form <username> @ <domainname>. There is, under the user point of view, an exception because shell service uses only <username>, so people can log in just as "umea" or "aemu".

It is also worth to mention that, since all of our services uses KERBEROS for authentication, all those services are configured in a way that let them automatically map between Kerberos Principals and this account name format.

  • <primary>/<instance> @ <REALM> maps to <username> @ <domainname>
  • or just <username>, case of shell service.

1.2 TTYrec videos playing

Please, just run ttyplay program and ttyrec file as argument: 

ttyplay /root/USER/TTYREC/<file>.ttyrec

You can get some control by using this three keybindings:

KeybindingAction
<space>jump to next event, don't wait.
0,1pause or set 1x speed.
Control-cexit.

… it could be enough, see "man ttyplay". Other player available is "ipbt" (reverse playing support, etc), see "man ipbt" if interested but ttyplay, as mentioned, is enough.

If screen gets corrupted without reverting to normal, your terminal size is less than 80x33. Please, resize it and play again.
